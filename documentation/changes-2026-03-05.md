# Great Games -- Change Log (2026-03-05)

Branch: `additionalStats`

This file captures changes implemented in this chat on 2026-03-05.
It is additive to `documentation/system.md` and `readme.md`.

## Summary

- Added a full period filter workflow for completed-game ranking.
- Added team-scoped filtering with MLB-aware options and non-MLB grouping.
- Added minimum-stars filtering to hide lower-rated games.
- Reworked the team selector into an expandable multi-select picker.
- Applied multiple UI polish passes for compactness, alignment, and responsive behavior.
- Added clickable game-details metadata links to SportsCult searches for ranked completed games.
- Finalized SportsCult search date token format as `DD MM YYYY` and kept links visually consistent with regular text.
- Updated box-score half-inning reveal to progressively unlock extra-inning columns only after the previous inning is fully revealed.
- Removed the extra-innings `innings-info` label from the scoreboard details panel.
- Replaced game-scoped pitcher highlights with pitcher-specific action photos that favor current-team uniforms.
- Simplified pitcher card stat lines to show only `Games` and `ERA` (removed `W-L`).
- Fixed same-day schedule loading so finalized games remain visible on the selected MLB day instead of disappearing from the list.
- Added mixed same-day ranking so completed games keep ranked star positions while live/upcoming games stay visible in the same table.

## Updates Implemented In This Chat

### 1) Period Filter Workflow

- Added an inline `Filter` button to the table toolbar next to `Game Date`.
- Added a new period filter modal with:
  - lookback days input,
  - team picker,
  - minimum stars control,
  - apply action.
- Added an active filter badge with remove action; clearing it restores default date-only view.
- Added period filter state handling so date changes re-apply the active filter automatically.

### 2) Date-Range Ranking + Table Context

- Implemented range fetch using `API.fetchGamesInRange(startDate, endDate)`.
- Applied ranking over completed games in the selected date window.
- Added optional team scoping and minimum-stars threshold on top of ranked results.
- Switched the 3rd table column from `Venue` to `Date` while filter mode is active.

### 3) Team Filtering Model (Final State)

- Team selector label now uses `All Teams` (replacing `All Games`).
- Team list defaults to MLB teams and includes an `Other` option.
- `Other` filters for games with at least one non-MLB team.
- Team selection supports single or multiple selections.

### 4) Team Picker UX (Final State)

- Replaced always-open native multi-select list with a collapsed, expandable picker.
- Expanded picker uses checkbox rows for multi-select behavior.
- Added outside-click and `Escape` close behavior for the expanded list.
- Added synchronization between hidden native select state and custom picker UI.

### 5) Minimum Stars Filter

- Added a minimum-stars selector to the period filter.
- Threshold is applied to ranked output using the same 1-5 star mapping used in rating display.
- Updated minimum-stars options to star-only labels in the dropdown.
- Active filter badge includes minimum-stars threshold summary.

### 6) Styling / Responsive Refinements

- Styled period filter controls to match existing modal components.
- Tuned team picker row density, checkbox sizing, and text alignment to avoid overlap.
- Reduced oversized picker elements and fixed label/checkbox column layout issues.
- Added responsive constraints for picker panel heights and control sizing.

### 7) Game Details Search Links (SportsCult)

- Converted `details-meta` text under `game-details-actions` into clickable links for ranked completed games.
- Added link generation using fixed search parameters:
  - `page=torrents`
  - `category=0`
  - `active=1`
  - `gold=0`
- Built search terms in this exact order:
  - `MLB`
  - game date
  - away team name
- Updated date search formatting from hyphenated form (`DD-MM-YYYY`) to tokenized form (`DD MM YYYY`) so generated links match SportsCult search expectations.
- Ensured links open in a new tab (`target="_blank"`) with `rel="noopener noreferrer"`.
- Removed underline styling from the link so it appears as regular details text while remaining clickable.

### 8) Box Score Extra-Inning Reveal Behavior (Final State)

- Added progressive extra-inning column reveal in half-inning mode.
- In extra-innings games, inning `10` appears only after the full `9th` inning is revealed.
- Inning `11` appears only after the full `10th` inning is revealed, and so on until game end.
- Added a dedicated hidden-column class in box-score rendering (`inning-column-hidden`) to hide locked inning columns.
- Removed the extra-innings `innings-info` label entirely from box-score output (no label is displayed in final UI).

### 9) Pitcher Highlight Image Validation

- Updated pitcher highlight-image matching to require MLB highlight taxonomy tags indicating pitching action.
- Excluded non-action/media-graphic assets (for example `data-visualization`) from pitcher image candidates.
- Limited candidates to video highlights so pitcher cards use actual in-game thumbnails when available.
- Kept existing fallback behavior: if no valid pitching-action highlight is found, the pitcher headshot remains in use.

### 10) Pitcher Action Image Source (Final State)

- Removed dependency on `/game/{gamePk}/content` highlight thumbnails for pitcher-card imagery.
- Switched pitcher-card images to MLB player action photos via `people/{pitcherId}/action/hero/current`, which is not tied to the current game.
- Added robust fallback flow for pitcher cards:
  - primary: player action image,
  - fallback 1: player headshot,
  - fallback 2: local unknown-player image asset.
- Result: pitcher cards prioritize in-action, on-field photos with current-team uniforms while avoiding mismatched same-game highlight selections.

### 11) Pitcher Stat Line Simplification

- Updated pitcher stat text in lineup pitcher cards to remove `W-L` output.
- Final pitcher stat format is now `Games` + `ERA` only (for example: `32 G, 2.91 ERA`).
- Applied consistently across both component paths used for pitcher card rendering.

### 12) Same-Day Completed Game Visibility Fix

- Adjusted selected-date routing so only dates strictly after the MLB schedule day use future-only parsing.
- Added a current-day mixed loader that combines:
  - completed (`Final`) games,
  - live games,
  - upcoming/scheduled games.
- Preserved schedule ordering so finalized games stay on the selected day list as statuses update from `In Progress` to `Final`.
- This resolves cases like `Czechia vs Chinese Taipei` on `2026-03-05`, where the game previously disappeared after completion.

### 13) Mixed Same-Day Ranking (Completed + Upcoming)

- For same-day mixed views, completed games are now ranked with current ranking-option weights.
- Rank values are shown only for completed games; live/upcoming entries remain visible with placeholder rank values.
- Table order remains schedule-driven so upcoming first-pitch context is preserved while completed games still receive stars/ranking.
