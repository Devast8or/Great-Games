### 7) Future-Game Data Availability Fallbacks (Follow-up 2026-03-05)

- Fixed stuck loading behavior for future games where starting lineups/pitcher stats are not yet published.
- Replaced indefinite lineup loading states with rendered placeholder table rows showing `Not Avaliable`.
- Kept lineup/pitcher layout containers visible when partial data fetches fail, so users still see full details structure.
- Hardened lineup extraction against missing `teams`, `battingOrder`, and player entries in future-game boxscore responses.
- Applied the same availability fallback behavior to both table-row details and legacy card details for consistency.

### 8) Spoiler-Safe Team Records (Follow-up 2026-03-05)

- Fixed matchup row records for completed games to avoid outcome spoilers from post-game standings values.
- Added completed-game pre-game record derivation in parser logic:
  - winning team record shows one fewer win than post-game `leagueRecord`.
  - losing team record shows one fewer loss than post-game `leagueRecord`.
- Added winner/loser resolution helper with score-first logic and `isWinner` fallback handling.
- Kept future/scheduled game record behavior unchanged.

### 9) Box Score Table Visual Refresh (Follow-up 2026-03-05)

- Redesigned the box score table to better match the current table/details visual language while preserving the baseball scoreboard feel.
- Increased emphasis for inning and `R/H/E` headers using stronger contrast and distinct header treatments.
- Added semantic class hooks to rendered box score headers/cells in `GameTableRow` for stable styling (`inning-header-cell`, `stat-header-cell`, `stat-cell`, `stat-break-cell`, `team-header-cell`, `inning-cell`).
- Enforced same-size scoring cells with CSS variables and fixed dimensions so cells remain uniform regardless of inning count.
- Kept extra-inning adaptability via horizontal scrolling in the score container instead of collapsing cell widths.
- Updated mobile breakpoints to scale the same box score cell-size variables for consistent sizing behavior on smaller screens.

### 10) Progressive Half-Inning Score Reveal (Follow-up 2026-03-05)

- Added a second score action button in table details for progressive reveal flow.
- Progressive button now starts with `Reveal Half Inning` and reveals one half inning per click (`Top`/`Bottom`).
- Full reveal button label is now `Reveal Full Score` in both table-row and legacy card paths.
- Removed restart copy (`Restart Half-Inning Reveal`) so completed state returns to the same `Reveal Half Inning` baseline label.
- Kept full reveal and half-inning reveal controls the same fixed size so label changes do not cause layout shift.
- Unrevealed inning cells remain masked until each half inning is revealed.
- `R/H/E` totals now update cumulatively as each half inning is revealed.
- Existing full reveal button remains available to instantly show/hide the full final box score.

### 11) Ranking Table Game Type Column (Follow-up 2026-03-05)

- Added a new `Type` column to the ranking table between `Venue` and `Rating`.
- Populated the column from MLB schedule `gameType` metadata already present in parsed game objects.
- Added game-type label formatting in table rows with user-friendly display values.
- Updated table details-row `colSpan` and responsive table width rules to account for the extra column.
- Expanded parser output to preserve schedule series metadata (`seriesDescription`, `seriesGameNumber`, `description`) for both completed and future games.
- Added `gameType` disambiguation logic so special-event `F` games with World Baseball Classic series metadata render as `WBC` instead of being mislabeled as Wild Card.

### 12) MLB Schedule Day Alignment + Live WBC Coverage (Follow-up 2026-03-05)

- Added MLB schedule timezone alignment on schedule requests via `timeZone=America/New_York` so date buckets match MLB.com scoreboard day boundaries.
- Updated date-mode decision in `UI` to compare selected date against MLB schedule-day "today" (America/New_York), not browser-local midnight.
- Reworked completed-game day filtering to use timezone-scoped schedule bucket date (`dates[].date`) as canonical played day; this avoids cross-midnight `officialDate` mismatches.
- Included `Live` games in the upcoming-day parser so in-progress games (for example, WBC pool play) are not dropped from the selected MLB day view.
- Updated status rendering for upcoming-day rows so live games display `In Progress`, while scheduled/preview rows display first-pitch time.
- Made the status column header conditional: `First pitch @` is shown only when all displayed upcoming-day rows are scheduled/preview; mixed or live slates use `Status`.
